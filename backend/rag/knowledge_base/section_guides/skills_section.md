# Skills Section Guide

## Purpose of the Skills Section

The Skills section serves two critical functions:
1. **ATS keyword matching** — This is where automated systems look for exact matches against job description requirements.
2. **Quick human scanning** — Recruiters glance at this section to confirm you have the baseline qualifications before reading further.

Place the Skills section **near the top** of your resume, typically after the Professional Summary and before Experience.

---

## Hard Skills vs. Soft Skills

### Hard Skills (Technical Skills)
Specific, teachable, measurable abilities. These are what ATS systems primarily scan for.

**Examples**: Python, SQL, Tableau, Adobe Photoshop, financial modeling, HIPAA compliance, AWS, Kubernetes, SolidWorks, Salesforce, A/B testing.

**Always include hard skills.** They are the backbone of ATS matching.

### Soft Skills
Interpersonal and behavioral abilities. These are harder to quantify and less useful in the Skills section.

**Examples**: Leadership, communication, problem-solving, teamwork, adaptability.

**Guideline**: Do NOT list generic soft skills in your Skills section. Instead, **demonstrate them through your experience bullets**:
- Instead of listing "Leadership" → write "Led a team of 12 engineers to deliver a $2M platform migration on time."
- Instead of listing "Communication" → write "Presented quarterly analytics reports to C-suite stakeholders, driving $500K budget reallocation."

**Exception**: Include a soft skill in the Skills section only when the job description explicitly lists it as a required qualification AND you have no other way to surface it.

## ATS-Friendly Formatting

### Do's ✅
- List skills as **comma-separated keywords** or in a **simple bulleted list**.
- Use the **exact terminology** from the job description.
- Include both **acronyms and full names**: "Search Engine Optimization (SEO)," "Amazon Web Services (AWS)."
- Group skills into **labeled categories** for readability.

### Don'ts ❌
- **No skill bars, graphs, or star ratings** — ATS cannot parse visual skill representations, and they convey no objective meaning.
- **No tables** for skills — some ATS parsers misread table content.
- **No vague terms** — "Microsoft Office" is acceptable; "Computer skills" is not.
- **No self-assessed proficiency levels** — "Python (Expert)" is subjective. Let your experience bullets demonstrate proficiency.

## Skill Categorization

Organizing skills into categories improves both readability and ATS matching. Use categories relevant to your field:

### For Software Engineers
```
Skills
Languages: Python, JavaScript, TypeScript, Go, SQL
Frameworks: React, Node.js, Django, FastAPI, Spring Boot
Cloud & DevOps: AWS (EC2, S3, Lambda), Docker, Kubernetes, Terraform, GitHub Actions
Databases: PostgreSQL, MongoDB, Redis, DynamoDB
Tools: Git, Jira, Datadog, Figma
```

### For Marketing Professionals
```
Skills
Digital Marketing: SEO, SEM, Google Ads, Meta Ads, LinkedIn Ads, A/B Testing
Analytics: Google Analytics 4, Tableau, Mixpanel, Looker, SQL
Content & CRM: HubSpot, Mailchimp, WordPress, Hootsuite, Canva
Methodologies: Marketing Automation, Lead Scoring, Attribution Modeling
```

### For Financial Analysts
```
Skills
Financial Analysis: Financial Modeling, DCF Valuation, LBO Modeling, Forecasting
Tools: Excel (Advanced — VLOOKUP, Pivot Tables, VBA), Bloomberg Terminal, Capital IQ, SAP
Data & Visualization: SQL, Python (Pandas), Tableau, Power BI
Regulatory: SOX Compliance, GAAP, IFRS
```

## How Many Skills to Include

- **10–20 skills** is the ideal range.
- Prioritize skills that appear in the job description.
- Remove skills that are irrelevant to the target role — even if you are proficient.
- Remove skills that are universally assumed (e.g., "Microsoft Word," "Email" — unless the job specifically requests them).

---

## Tailoring Skills for Each Application

1. **Read the job posting** and highlight every skill, tool, platform, and methodology mentioned.
2. **Match your skills** to the posting — use their exact language.
3. **Reorder** your skills so the most relevant ones appear first within each category.
4. **Add missing keywords** only if you genuinely possess the skill. Never fabricate skills — you will be tested in interviews.

### Example of Tailoring
**Job posting requires**: React, TypeScript, GraphQL, AWS, CI/CD, agile

**Before tailoring**: Languages: Python, Java, JavaScript | Tools: Git, Docker
**After tailoring**: Languages: TypeScript, JavaScript, Python | Frameworks: React, GraphQL | Cloud & DevOps: AWS (Lambda, S3), Docker, CI/CD (GitHub Actions) | Methodologies: Agile (Scrum)

## Skills Section Placement by Role Type

| Role Type               | Placement                          |
|--------------------------|-------------------------------------|
| Technical / Engineering  | Right after Summary (critical for ATS) |
| Business / Management    | After Summary or after Experience   |
| Creative / Design        | After Summary; complement with portfolio link |
| Entry-Level / Student    | Right after Summary (compensates for limited experience) |
